# Welcome to WotLK Extensions

**WotLK Extensions** is a native DLL extension framework for **World of Warcraft 3.3.5a (Wrath of the Lich King)** focused on expanding the capabilities of the original client through modern runtime enhancements.

The project provides a foundation for extending the 3.3.5a game client beyond its original limitations, enabling developers to implement custom systems, graphical enhancements, runtime hooks, client-side features, and experimental functionality without modifying the core game executable directly.

## Purpose

World of Warcraft 3.3.5a remains one of the most widely used versions for emulation and custom server development, but its client architecture has many hardcoded limitations. **WotLK Extensions** exists to provide a flexible and modular way to extend that client through native code injection and runtime systems.

This framework is intended for developers who want to:

- Extend client functionality through DLL-based systems
- Hook and modify client behavior at runtime
- Build custom graphical or UI enhancements
- Add new client-side systems and experimental features
- Prototype advanced modifications for private server development
- Research and explore the WoW 3.3.5a client architecture

## Project Goals

WotLK Extensions is designed with several core goals in mind:

- **Modularity** – Build reusable extension systems without tightly coupling changes to the client
- **Runtime Flexibility** – Enable feature development through hooks, injected systems, and dynamic behavior
- **Developer-Focused Tooling** – Provide a practical foundation for experimentation and advanced client engineering
- **Maintainability** – Keep extension logic organized, clean, and scalable
- **Modern Development Practices** – Apply structured native development techniques to legacy game client modding

## Scope

WotLK Extensions is not a standalone game client or emulator. Instead, it acts as an **extension layer** for the WoW 3.3.5a client, allowing developers to implement functionality that would otherwise require direct executable patching or invasive client modification.

Potential use cases include:

- Client-side rendering extensions
- Native hook systems
- Custom UI integrations
- Debugging and developer tooling
- Experimental game systems
- Runtime behavior modification
- Research and reverse engineering projects
